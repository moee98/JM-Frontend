A small car modification shop was running their entire business on spreadsheets and WhatsApp messages. I built them a custom dashboard — here's what changed.

Before: jobs were tracked in a spreadsheet, invoices were typed up manually, expenses lived in a notebook, and nothing talked to anything else. Errors crept in. Time was wasted. Things got missed.

After: one system handles it all.

**What I built:**
→ Work orders from creation to payment — all in one place
→ Customer & vehicle records with inspection forms
→ Invoices with VAT, generated and emailed in one click
→ Expense tracking with categorisation and file attachments
→ Split payments across multiple methods
→ Calendar scheduling for jobs
→ eBay and Square integrations

The result? Less time on admin, fewer mistakes, and everything organised in one place instead of scattered across five different tools.

**Tech behind it:**
- **Frontend**: React 19 + TypeScript + Vite + Tailwind CSS 4 + TanStack Query + Zustand
- **Backend**: ASP.NET Core (.NET 10) + Entity Framework Core + SQL Server
- **Auth**: JWT + HttpOnly cookies + role-based access
- **Deployment**: Docker Compose + Nginx, self-hosted on a local Windows server

It's open source — happy to chat about the architecture or how it was built.
